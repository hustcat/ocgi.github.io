Features

Application Interactive Update

Support application interactive update, delete the replica only after the application confirmed

In-place Update

Support in-place update image, and the Pod will not be recreated. The local cache data GameServer can be retained when updating

General Pod Auto-scaling

Support multiple modes (resource indicators/custom indicators/timing/events/webhook) Pod scaling strategies

Application-defined Scaling-down Order

Application can define the scaling-down order of replicas, application can choose replica with least user-accessed to delete

Better Cluster Auto-scaling

It can be seamlessly integrated with the cluster auto-scaling. Can choose any replica to delete after application confirmed when cluster scaling-down

Opensource

All components are built on K8s, completely free and open source, and can run on any K8s cluster
